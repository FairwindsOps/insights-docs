# Membership

For each organization, there are two levels of access:

  • members can view any data in the Insights dashboard, but cannot make changes
  • admins can make changes, install the Insights Agent, and access the API

# Managing Members

To add a new user to your organization, go to your organization's settings page. You can then enter an email address, and decide what their level of access should be.

Once a user has been added to the organization, you can edit their access by clicking the icon next to their name

# Namespace Restrictions

Particular namespaces can be hidden from the UI for member users. This is helpful if members don't need to see issues e.g. related to kube-system.

To hide a particular namespace from the UI, you can add the following annotation to it:

insights.fairwinds.com/adminOnly=true

Note that this will not prevent member users from accessing data about those namespaces - they will still be able to view the raw reports, which may contain information about those namespaces. These namespaces will only be hidden from the Action Items table and other visual interfaces.
